1. read $ARGUMENTS and pick a single top level todo from there, it must be the most important unfinished top level item (1 bug or 1 feature) - it can be multiple nested todos under the 1 top level todo
2. Continue iterating until done. update the file as needed
3. Test to make sure your implementation works as intended
4. Once done, mark the todo as complete and then commit your changes using the `gai` command line (it's a special alias for `git commit -m`)
